# 04. PQ — Product Quantization (Jégou 2011)

## 🎯 핵심 질문

- Subvector 분해 $x = (x_1, \ldots, x_m)$ 로 왜 compression 이 가능한가?
- 각 subvector 에 k-means 를 독립적으로 적용하면 $k^m$ 개의 possible codes 가 나오는데, 이게 원본과 얼마나 다른가?
- ADC (Asymmetric Distance Computation) 는 뭔가?
- IVF-PQ 의 메모리 효율이 정말 235× 수준인가?

---

## 🔍 왜 PQ 는 단순하면서도 강력한가

Jégou et al. (2011) 의 Product Quantization 은 **각 벡터를 매우 작은 코드로 압축** 하면서도 거리 계산은 원본 벡터에 가깝게 수행. 핵심은 "덧셈의 분해" (product): $d$-dimensional 벡터를 $m$ 개의 $d/m$-dimensional subvectors 로 쪼개고, 각각 **독립적 k-means quantization**. 

결과: 단 $m \log_2 k$ bits 로 원본 벡터 표현. 예: $d=768, m=96, k=256$ → 96×8 bits = 768 bits (FP32 대비 32배 압축).

---

## 📐 수학적 선행 조건

- Vector quantization 개념 (codebook, codeword)
- k-means clustering (각 subspace 에서 독립 실행)
- Asymmetric distance metric
- 정보 이론: rate-distortion tradeoff

## 📖 직관적 이해

### Subvector Decomposition

```
원본 벡터 x ∈ ℝ^d (e.g., d=768)
   │
   ├─ Split into m parts (e.g., m=96)
   │  x = (x^(1), x^(2), ..., x^(96))
   │  각 x^(j) ∈ ℝ^8  (subvector dimension = 768/96 = 8)
   │
   └─ Quantize each independently
      · Subvector 1: k-means with k=256 → code c₁ ∈ [0, 255]
      · Subvector 2: k-means with k=256 → code c₂ ∈ [0, 255]
      · ...
      · Subvector 96: k-means with k=256 → code c₉₆ ∈ [0, 255]
   
결과: 원본 벡터 = 96개의 8-bit codes
메모리: 96 × 8 bits = 768 bits (원본: 768 × 32 bits)
압축률: 32배

코드북: 각 subspace j 마다 (k × d/m) floating-point 저장
        총: m × k × (d/m) = k × d floats (원본과 비슷)
```

### k-means Per Subspace

```
Subspace j 를 위한 centroid 학습:
  C^(j) = {c^(j)_0, c^(j)_1, ..., c^(j)_{k-1}} ⊂ ℝ^(d/m)
  
각 x^(j) 에 대해 nearest centroid index 저장:
  code_j = argmin_i ||x^(j) - c^(j)_i||²

검색 시 (ADC):
  distance(q, x) ≈ ||q - \tilde{x}||²
  = Σⱼ ||q^(j) - c^(j)_{code_j}||²

핵심: reconstruction x̃ 없이 거리 계산 가능 (memory efficient)
```

### Accuracy vs Compression Trade-off

```
Compression Ratio ▲
        32×│●────────
           │
        16×│    ●
           │       ●
         8×│          ●
           │
         4×│              ●
           │
         1×│                  ●
           └─────────────────────► Recall@10
             80%  85%  90%  95% 99%
             
Typical: m=96, k=256 → 32× compression, 90-95% recall
```

## ✏️ 엄밀한 정의

### 정의 4.1 — Product Quantization

**입력**: 벡터 $x \in \mathbb{R}^d$

**Decomposition**: $x = (x^{(1)}, \ldots, x^{(m)})$ where $x^{(j)} \in \mathbb{R}^{d/m}$.

**Quantization**: For each $j = 1, \ldots, m$:
$$\tilde{x}^{(j)} = \arg\min_{c \in C^{(j)}} \|x^{(j)} - c\|^2$$
where $C^{(j)} = \{c^{(j)}_0, \ldots, c^{(j)}_{k-1}\}$ (codebook for subspace $j$).

**Code**: $\mathbf{code}(x) = (\text{idx}_1, \ldots, \text{idx}_m) \in [0, k-1]^m$ where $\text{idx}_j = \arg\min_i \|x^{(j)} - c^{(j)}_i\|$.

**Reconstruction** (optional): $\tilde{x} = (\tilde{x}^{(1)}, \ldots, \tilde{x}^{(m)})$.

**Compression**: From $d \cdot 32$ bits (FP32) to $m \cdot \log_2 k$ bits.

### 정의 4.2 — Asymmetric Distance Computation (ADC)

**Goal**: distance(query, quantized database vector) without reconstructing $\tilde{x}$.

**Method**: 
$$d_{\text{ADC}}(q, \mathbf{code}(x)) = \sum_{j=1}^m \|q^{(j)} - c^{(\mathbf{code}(x))_j}^{(j)}\|^2$$

where $c^{(\mathbf{code}(x))_j}^{(j)}$ = centroid of subspace $j$ corresponding to code index.

**Approximation**: This is exact if $x = \tilde{x}$ (full reconstruction), otherwise approximation.

### 정의 4.3 — Storage and Lookup Complexity

**Database Storage**:
- Codes: $N \cdot m \cdot \log_2 k$ bits
- Codebooks: $m \cdot k \cdot (d/m) \cdot 32 = k \cdot d \cdot 32$ bits (typically negligible vs database)

**Query Computation (ADC)**:
- Precompute: $m \cdot k$ distances (query vs all centroids) = $O(m \cdot k \cdot (d/m)) = O(k \cdot d)$ = O(d) with precomputation
- Lookup per candidate: $O(m)$ (sum of m precomputed distances)

---

## 🔬 정리와 증명

### 정리 4.1 — Quantization Error Bound

PQ quantization error $\mathcal{E} = \|x - \tilde{x}\|^2$:
$$
\mathcal{E} = \sum_{j=1}^m \|x^{(j)} - c^{(j)}_{\text{idx}_j}\|^2 \leq m \cdot \sigma^2(x^{(j)})
$$

where $\sigma^2(x^{(j)})$ = average quantization error in subspace $j$ (depends on k-means quality).

**증명**: Each subspace quantization contributes independently. For subspace $j$ with codebook size $k$:
$$
\|x^{(j)} - \tilde{x}^{(j)}\|^2 \leq \sigma^2(x^{(j)})
$$

Summing over $m$ subspaces bounds total error. $\square$

### 정리 4.2 — Number of Possible Codes

With $m$ subspaces and $k$ codewords each:
$$
|\text{codebook}| = k^m
$$

**Implication**: 
- $m=96, k=256$: $256^{96}$ possible codes (astronomically large)
- But only $N$ are actually stored (where $N$ = database size)
- This "overcompleteness" (exponentially more codes than samples) justifies call "Product Quantization"

**증명**: Each of $m$ subspaces independently chooses 1 of $k$ codewords $\square$

### 정리 4.3 — ADC Distance Approximation Quality

Let $\mathcal{E}_j = \|x^{(j)} - \tilde{x}^{(j)}\|$ (per-subspace error). Then:
$$
|d_{\text{ADC}}(q, x) - d(q, \tilde{x})| \leq 2\|q\| \cdot \sqrt{\sum_j \mathcal{E}_j^2}
$$

(증명은 triangle inequality 활용)

**의미**: Query 가 작을수록 (정규화된 벡터), quantization error 의 영향이 커짐. 하지만 일반적으로 bounded.

## 💻 Python / FAISS / NumPy 구현 검증

### 실험 1 — Manual PQ Codebook 학습

```python
import numpy as np
from sklearn.cluster import KMeans

# Dataset
N, d = 100_000, 768
X = np.random.randn(N, d).astype(np.float32)
X = X / np.linalg.norm(X, axis=1, keepdims=True)

# PQ parameters
m = 96  # number of subspaces
k = 256  # codebook size per subspace
d_sub = d // m  # = 8

# Learn codebooks
codebooks = []
codes = np.zeros((N, m), dtype=np.uint8)

for j in range(m):
    # Extract subvector j from all database vectors
    X_sub = X[:, j*d_sub:(j+1)*d_sub]  # (N, d_sub)
    
    # k-means on subspace j
    kmeans = KMeans(n_clusters=k, n_init=5, random_state=42)
    codes[:, j] = kmeans.fit_predict(X_sub)
    codebooks.append(kmeans.cluster_centers_)  # (k, d_sub)

print(f"Codebooks learned. Codes shape: {codes.shape}")
print(f"Codebook sizes: {[cb.shape for cb in codebooks]}")

# Storage size
original_bits = N * d * 32
pq_bits = N * m * np.log2(k)
compression = original_bits / pq_bits
print(f"Original: {original_bits/1e9:.1f} GB, PQ: {pq_bits/1e9:.1f} GB, "
      f"Compression: {compression:.1f}×")
```

### 실험 2 — Asymmetric Distance Computation

```python
def pq_distance_adc(q, codes, codebooks, X_orig=None):
    """
    Compute distance(q, quantized X) without reconstruction.
    
    Args:
        q: query vector (d,)
        codes: database codes (N, m)
        codebooks: list of m codebooks, each (k, d/m)
        X_orig: (optional) original database for verification
    """
    N, m = codes.shape
    d_sub = q.shape[0] // m
    distances = np.zeros(N)
    
    # For each subspace, precompute distances from q to all centroids
    for j in range(m):
        q_sub = q[j*d_sub:(j+1)*d_sub]  # (d_sub,)
        cb_j = codebooks[j]  # (k, d_sub)
        
        # Distance from q_sub to all k centroids
        dists_j = np.linalg.norm(cb_j - q_sub, axis=1) ** 2  # (k,)
        
        # Look up distances for each database code
        for i in range(N):
            code_idx = codes[i, j]
            distances[i] += dists_j[code_idx]
    
    return distances

# Test
q = X[0]
distances_adc = pq_distance_adc(q, codes, codebooks)
top_10_adc = np.argsort(distances_adc)[:10]

# Verify against exact distance (on original vectors)
distances_exact = np.linalg.norm(X - q, axis=1) ** 2
top_10_exact = np.argsort(distances_exact)[:10]

recall = len(set(top_10_adc) & set(top_10_exact)) / 10
print(f"Recall@10 with PQ: {recall:.1%}")
print(f"Top-10 exact: {top_10_exact}")
print(f"Top-10 PQ ADC: {top_10_adc}")
```

### 실험 3 — Reconstruction and Error Quantification

```python
def pq_reconstruct(codes, codebooks, m, d_sub):
    """Reconstruct vectors from codes"""
    N = codes.shape[0]
    X_reconstructed = np.zeros((N, m * d_sub), dtype=np.float32)
    
    for j in range(m):
        for i in range(N):
            code_idx = codes[i, j]
            X_reconstructed[i, j*d_sub:(j+1)*d_sub] = codebooks[j][code_idx]
    
    return X_reconstructed

X_recon = pq_reconstruct(codes, codebooks, m, d_sub)

# Quantization error
quantization_error = np.linalg.norm(X - X_recon) ** 2 / N
print(f"Average quantization error: {quantization_error:.4f}")

# Verify: distance error under reconstruction
distances_recon = np.linalg.norm(X_recon - X[0], axis=1) ** 2
recall_with_recon = len(set(np.argsort(distances_recon)[:10]) & 
                        set(np.argsort(np.linalg.norm(X - X[0], axis=1) ** 2)[:10])) / 10
print(f"Recall@10 with reconstruction: {recall_with_recon:.1%}")
```

### 실험 4 — FAISS IVF-PQ Index

```python
try:
    import faiss
except ImportError:
    print("Install: pip install faiss-cpu")
    exit(1)

# Build IVF-PQ index
d = 768
nlist = 100  # number of clusters for IVF
m = 96       # number of subspaces for PQ
nbits = 8    # bits per subspace code (k = 2^8 = 256)

# Create quantizer for IVF
quantizer = faiss.IndexFlatL2(d)

# Create IVF-PQ index
index = faiss.IndexIVFPQ(quantizer, d, nlist, m, nbits)

# Train: need to train both IVF (clustering) and PQ (subspace quantizers)
index.train(X)

# Add vectors
index.add(X)

# Query
q = X[0:1]
index.nprobe = 10  # how many clusters to search

D, I = index.search(q, k=10)
print(f"IVF-PQ search results:")
print(f"Distances: {D[0]}")
print(f"Indices: {I[0]}")

# Verify index integrity
print(f"Index ntotal: {index.ntotal}")
print(f"Index is_trained: {index.is_trained}")
```

## 🔗 실전 활용

| 상황 | PQ 추천 | 이유 |
|------|--------|------|
| 100M-1B vectors, limited RAM | Yes (IVF-PQ) | 32배 압축 → memory-efficient |
| Dense embedding + filtering | Yes | ADC 빠르고 메모리 효율 |
| Real-time re-indexing | No | Codebook 재학습 비용 |
| GPU 기반 search | Yes | FAISS GPU 지원 |
| Commodity server (128GB RAM) | Strongly Yes | 32GB embedding → IVF-PQ with 1GB memory |
| Streaming data insert | Somewhat | Codebook 고정 → 새 data 양자화만 |

---

## ⚖️ 가정과 한계

- **Independence of subspaces**: 각 subspace 의 quantization error 가 independent 하다고 가정 — 실제로는 상관성 있음 (embedding 의 structure).
- **Balanced subspace clustering**: 각 codebook size k 가 같다고 가정 — 최적화는 subspace 별로 다른 k 사용 가능 (복잡).
- **Fixed subspace boundaries**: subvector 의 차원 결정 후 고정 — 차원 재배치 (PCA 등) 로 개선 가능 (OPQ: Optimized Product Quantization).
- **ADC 근사**: query-centroid 거리로 query-database 거리 근사 — reconstruction 과의 차이 무시.
- **No learning updates**: 학습 후 codebook 고정 — 새로운 data distribution 에 적응 불가.

---

## 📌 핵심 정리

$$\boxed{\text{compression} = \frac{d \cdot 32}{m \cdot \log_2 k} = \frac{d \cdot 32}{m \cdot 8} = \frac{d}{m} \text{ (for } k=256\text{)}}$$

**예시**: $d = 768, m = 96, k = 256$ → $768/96 = 8$ bytes/vector (32× compression)

| 방면 | 값 |
|-----|-----|
| Codebooks | m × k × (d/m) floats |
| Database codes | N × m bits (or uint8 per code) |
| Query-centroid precomp | m × k distances (O(d)) |
| Per-candidate distance | O(m) lookups |
| Recall@10 typical | 90-95% (with IVF tuning) |

**IVF-PQ 특징**:
- **Coarse stage**: IVF 로 candidate 선택
- **Fine stage**: PQ-ADC 로 정확한 거리 계산
- **Memory**: $(K \cdot d) + (N \cdot m)$ bytes (K = # IVF clusters)
- **100M vectors, d=768, m=96**: 1.2 GB (vs 307 GB original)

> **핵심**: PQ 는 단순하지만 효과적. IVF-PQ 의 조합이 현대 벡터 데이터베이스의 backbone.

## 🤔 생각해볼 문제 (+ 해설)

**문제 1 (기초)**: m=96, k=256 일 때 각 subspace 는 몇 차원이고, 각 code 는 몇 bit 인가?

<details>
<summary>해설</summary>

$d/m = 768/96 = 8$ 차원. $\log_2(256) = 8$ bit. 총: $m \cdot 8 = 96 \cdot 8 = 768$ bits = 96 bytes (FP32 3,072 bytes 대비 32배 압축).

</details>

**문제 2 (심화)**: ADC 거리 계산 시 query 와 centroid 간 거리를 precompute 하는 이유는?

<details>
<summary>해설</summary>

**Precompute benefit**: 모든 N 개 database code 에 대해 query-centroid 거리 사용 (재계산 X).

Step 1: Precompute (한 번만)
- For each subspace j: compute distances from query q^(j) to all k centroids → m×k distances.

Step 2: Lookup (반복)
- For each database code, lookup precomputed distances and sum (O(m) per vector).

**Without precomputation**: 각 database 벡터마다 m subspace, 각 subspace마다 centroid 거리 계산 → O(N·m·d/m) = O(N·d). 즉, linear scan 처럼 느려짐.

**Precomputation**: O(m·k·d/m) + O(N·m) = O(k·d + N·m). N 이 크면 O(N·m) ≈ linear in N but with small constant m.

</details>

**문제 3 (논문 비평)**: PQ 논문 (Jégou 2011) 에서 제안한 OPQ (Optimized PQ) 는 뭔가? 왜 필요?

<details>
<summary>해설</summary>

OPQ: subspace 의 경계를 고정하지 않고, **학습 데이터를 rotation** 시켜 subspace 간 상관성을 제거한 후 PQ 적용. 

**동기**: 원본 embedding 이 특정 방향에 correlation 을 가질 수 있음 (예: PCA 축을 따라 분산 집중). PQ 는 independent subspace 가정 → 성능 저하. OPQ 는 rotation R 을 학습하여:

1. Y = R·X (rotate data)
2. PQ on Y (각 subspace 가 더 독립적)

**효과**: 같은 m, k 로 더 높은 recall (2-3% 개선 typical). 

**비용**: Rotation 학습은 추가 computation. 하지만 one-time offline cost. FAISS 에서도 지원 (`faiss.OPQMatrix`).

</details>
