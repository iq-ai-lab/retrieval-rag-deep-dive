# 03. BM25 — Probabilistic Relevance Framework (Robertson 1995)

## 🎯 핵심 질문

- TF-IDF 의 선형 TF 가정이 왜 부정확한가? "term frequency saturation" 이 정확히 무엇인가?
- 2-Poisson Eliteness Model 에서 어떻게 BM25 의 공식이 유도되는가?
- $k_1$ (saturation parameter) 와 $b$ (length normalization) 는 정확히 무엇을 제어하는가?
- BM25 가 여전히 BEIR benchmark 에서 강력한 이유는?

---

## 🔍 왜 BM25 가 retrieval·RAG 에 중요한가

BM25 는 1995년 Robertson et al. 이 "Okapi project" 에서 제시한 후, 30년 가까이 **가장 강력한 sparse retrieval baseline** 으로 군림하고 있습니다. 비록 dense embedding 이 등장했지만, BM25 는 다음 이유로 필수적입니다.

1. **이론적 근거의 명확성** — 확률론 (Probabilistic Relevance Framework) 에 기반. TF-IDF 의 경험적 가정을 수학적으로 정당화.
2. **Zero-shot 강력함** — 학습 데이터 없이도 작동. Fine-tuning 불가능한 도메인에서 유일한 선택지.
3. **Hybrid RAG 의 기준** — Dense + BM25 hybrid 는 현재도 SOTA. Dense 만으로 실패 시 BM25 추가로 recall 증가.
4. **BEIR benchmark 의 우월성** — 0-shot dense 모델 (DPR, Contriever) 을 능가하는 경우 많음 (Thakur et al. 2021).
5. **해석 가능성** — Dense embedding 과 달리 각 term 의 기여를 정확히 계산 가능.

## 📖 직관적 이해

### 왜 "Saturation" 이 필요한가

```
TF-IDF: score ∝ TF × IDF
  문서에서 "machine" 이 1번 나타남:  score = 1 × IDF = 1.0 × IDF
  문서에서 "machine" 이 100번:       score = 100 × IDF = 100 × IDF
  → 100배 차이 (비현실적! 실제로는 2~3배 정도만 더 relevant)

BM25: Saturation 적용
  문서에서 "machine" 이 1번:  score ≈ 0.5 × IDF
  문서에서 "machine" 이 100번: score ≈ 1.0 × IDF  (거의 포화)
  → 합리적인 gradient (diminishing return)
```

### 2-Poisson Eliteness Model 의 직관

```
Assumption: 각 document 는 두 "state" 중 하나
  - Elite (relevant to term t): Poisson(λ₁) 로 term t 발생
  - Non-elite (not relevant):  Poisson(λ₀) 으로 term t 발생

예시 (영어 "the"):
  Elite document (관련):      E[tf] = λ₁ = 10 (자주 나타남)
  Non-elite document:         E[tf] = λ₀ = 2  (가끔 나타남)

BM25 는 문서가 elite 일 posterior probability 를 계산:
  P(elite | tf) ∝ P(tf | elite) × P(elite) = Poisson(tf; λ₁) / Poisson(tf; λ₀)
```

## ✏️ 엄밀한 정의

### 정의 3.1 — Probabilistic Relevance Framework (PRF)

**Relevance** $R$ 는 binary random variable: $R = 1$ (relevant), $R = 0$ (non-relevant).

**Ranking principle**: Document $d$ 와 query $q$ 의 relevance score:

$$\text{Score}(d,q) = \log \frac{P(R=1|d,q)}{P(R=0|d,q)}$$

(Log-odds of relevance, called "retrieval status value" or RSV)

Using Bayes:

$$\text{Score}(d,q) = \log\frac{P(d,q|R=1)P(R=1)}{P(d,q|R=0)P(R=0)}$$

Assuming prior $P(R=1) = P(R=0) = 0.5$:

$$\text{Score}(d,q) = \log\frac{P(d,q|R=1)}{P(d,q|R=0)}$$

### 정의 3.2 — 2-Poisson Eliteness Model

각 query term $t$ 에 대해, document 는 두 가지 상태 중 하나:
- **Elite** ($E_t = 1$): term $t$ 와 relevant 한 문서
- **Non-elite** ($E_t = 0$): term $t$ 와 무관한 문서

**Assumption**: Document 에서 term $t$ 의 frequency는 Poisson distribution:

$$\text{TF}(t,d) | E_t = e \sim \text{Poisson}(\lambda_e)$$

여기서 $\lambda_1$ (elite) > $\lambda_0$ (non-elite).

**Independence**: 각 term 의 relevance 는 독립적.

### 정의 3.3 — BM25 (Best Matching 25)

Query $q = \{t_1, \ldots, t_k\}$ (query terms) 에 대한 document $d$ 의 score:

$$\text{BM25}(d,q) = \sum_{t \in q} \text{IDF}(t) \cdot \frac{\text{TF}(t,d) \cdot (k_1 + 1)}{\text{TF}(t,d) + k_1 \cdot (1 - b + b \cdot \frac{|d|}{\text{avgdl}})}$$

**Parameters**:
- $\text{IDF}(t) = \log\left(\frac{N - \text{df}(t) + 0.5}{\text{df}(t) + 0.5}\right)$ (modified from TF-IDF)
- $k_1 \in [1.2, 2.0]$: term frequency saturation. 작을수록 빨리 포화.
- $b \in [0, 1]$: length normalization. $b=0$ 이면 length normalization 없음, $b=1$ 이면 full normalization.
- $|d|$: document length (term count)
- $\text{avgdl}$: average document length in collection

---

## 🔬 정리와 증명

### 정리 3.1 — 2-Poisson 모델에서 BM25 유도

**Setting**: Document $d$ 에서 term $t$ 의 frequency 가 Poisson($\lambda_e$) 로 분포. Relevance 여부를 결정하는 "eliteness" $E_t \in \{0, 1\}$.

**Likelihood ratio**:

$$\frac{P(\text{TF}=x|E_t=1)}{P(\text{TF}=x|E_t=0)} = \frac{\text{Poisson}(x; \lambda_1)}{\text{Poisson}(x; \lambda_0)} = \left(\frac{\lambda_1}{\lambda_0}\right)^x \cdot \exp(\lambda_0 - \lambda_1)$$

Log-odds:

$$\log\frac{P(E_t=1|\text{TF}=x)}{P(E_t=0|\text{TF}=x)} = x \log\frac{\lambda_1}{\lambda_0} + \log\frac{P(E_t=1)}{P(E_t=0)} - (\lambda_1 - \lambda_0)$$

Letting $\alpha = \log(\lambda_1/\lambda_0)$ 와 $\beta = \log(P(E_t=1)/P(E_t=0)) - (\lambda_1 - \lambda_0)$:

$$\text{RSV}_t = \alpha \cdot x + \beta$$

이를 다시 정리하면, $k_1$ 는 saturation 을 제어하는 parameter 로:

$$\text{RSV}_t \approx \log\frac{\lambda_1}{\lambda_0} \cdot \frac{x(k_1+1)}{x+k_1}$$

(intuitive: $x \to 0$ 일 때 0에 수렴, $x \to \infty$ 일 때 $\log(\lambda_1/\lambda_0)$ 에 수렴)

**Length normalization** ($1-b+b \cdot |d|/\text{avgdl}$) 은 document length bias 제거.

최종 BM25 공식은 이 유도에서 empirical 조정으로 얻어짐. $\square$

### 정리 3.2 — BM25 의 TF 포화 성질

BM25 의 TF 항:

$$f(x) = \frac{x(k_1+1)}{x+k_1}$$

는 단조증가 하지만 sublinear (saturation 있음):

1. $f(0) = 0$
2. $f(\infty) = k_1 + 1$
3. $f'(x) = \frac{k_1(k_1+1)}{(x+k_1)^2} > 0$ (increasing)
4. $f''(x) = -\frac{2k_1(k_1+1)}{(x+k_1)^3} < 0$ (concave, saturation)

**의미**: $x$ 가 클수록 $f'(x)$ 는 감소 → diminishing return. $\square$

### 정리 3.3 — Length Normalization Parameter $b$

BM25 에서 length normalization 항:

$$L(d) = 1 - b + b \cdot \frac{|d|}{\text{avgdl}}$$

- $b = 0$: No length normalization ($L(d) = 1$ for all $d$)
- $b = 1$: Full normalization ($L(d) = |d|/\text{avgdl}$, TF 가 document length 에 무관)
- $0 < b < 1$: Partial normalization (보통 $b = 0.75$)

이는 empirically optimal 이며, 너무 aggressive 하면 오히려 성능 저하 (매우 짧은 문서가 비정상적으로 높은 점수를 받을 수 있음).

## 💻 Python / Rank-BM25 / Elasticsearch 구현 검증

### 실험 1 — BM25 바닥부터 구현

```python
import numpy as np
from collections import defaultdict
import math

class BM25Retriever:
    """BM25 직접 구현"""
    
    def __init__(self, documents, k1=1.5, b=0.75):
        """
        documents: list of strings
        k1: term frequency saturation (default 1.5)
        b: length normalization (default 0.75)
        """
        self.k1 = k1
        self.b = b
        self.documents = documents
        self.n_docs = len(documents)
        
        # Tokenize documents
        self.tokenized_docs = []
        for doc in documents:
            self.tokenized_docs.append(doc.lower().split())
        
        # Vocabulary & document frequency
        self.vocab = set()
        self.doc_freq = defaultdict(int)
        
        for doc_tokens in self.tokenized_docs:
            unique_terms = set(doc_tokens)
            self.vocab.update(unique_terms)
            for term in unique_terms:
                self.doc_freq[term] += 1
        
        self.vocab = sorted(list(self.vocab))
        
        # Average document length
        self.avgdl = sum(len(doc) for doc in self.tokenized_docs) / self.n_docs
    
    def _compute_idf(self, term):
        """Modified IDF for BM25"""
        df = self.doc_freq.get(term, 0)
        # Robertson's BM25 IDF with smoothing
        idf = math.log((self.n_docs - df + 0.5) / (df + 0.5))
        return max(idf, 0)  # IDF >= 0
    
    def _compute_tf(self, term, doc_tokens):
        """Raw term frequency"""
        return doc_tokens.count(term)
    
    def score_document(self, query_tokens, doc_tokens):
        """
        BM25 score for (query, document)
        """
        score = 0.0
        doc_length = len(doc_tokens)
        
        for term in query_tokens:
            idf = self._compute_idf(term)
            tf = self._compute_tf(term, doc_tokens)
            
            # BM25 formula
            numerator = tf * (self.k1 + 1)
            denominator = tf + self.k1 * (1 - self.b + self.b * (doc_length / self.avgdl))
            
            score += idf * (numerator / denominator)
        
        return score
    
    def retrieve(self, query, k=None):
        """
        Query → top-k documents
        Returns: [(doc_idx, score), ...]
        """
        query_tokens = query.lower().split()
        
        scores = []
        for doc_idx, doc_tokens in enumerate(self.tokenized_docs):
            score = self.score_document(query_tokens, doc_tokens)
            scores.append((doc_idx, score))
        
        # Sort by score descending
        scores.sort(key=lambda x: x[1], reverse=True)
        
        if k is not None:
            scores = scores[:k]
        
        return scores

# 테스트
docs = [
    "machine learning basics",
    "deep learning neural networks",
    "machine learning algorithms",
    "natural language processing",
    "machine learning is fun to learn",
]

bm25 = BM25Retriever(docs, k1=1.5, b=0.75)
results = bm25.retrieve("machine learning", k=3)

print("BM25 retrieval results:")
for doc_idx, score in results:
    print(f"  Doc {doc_idx}: '{docs[doc_idx]}' (BM25: {score:.4f})")
```

### 실험 3 — $k_1$ (saturation) 와 $b$ (length norm) 의 효과 분석

```python
# 파라미터 변화에 따른 점수 변화

test_doc = ["machine"] * 10  # "machine" 이 10번 나타남
query = ["machine"]
avgdl = 5.0

def bm25_score(tf, k1, b, idf=1.0):
    """Simplified BM25 계산"""
    numerator = tf * (k1 + 1)
    denominator = tf + k1 * (1 - b + b * (len(test_doc) / avgdl))
    return idf * (numerator / denominator)

print("Effect of k1 (saturation parameter):")
print(f"{'k1':<8} {'TF=1':<10} {'TF=5':<10} {'TF=10':<10} {'TF=20':<10}")
for k1 in [0.5, 1.0, 1.5, 2.0, 3.0]:
    scores = [bm25_score(tf, k1, 0.75) for tf in [1, 5, 10, 20]]
    print(f"{k1:<8.1f} {scores[0]:<10.4f} {scores[1]:<10.4f} {scores[2]:<10.4f} {scores[3]:<10.4f}")

print("\nEffect of b (length normalization parameter):")
print(f"{'b':<8} {'Score':<10}")
for b in [0.0, 0.25, 0.5, 0.75, 1.0]:
    score = bm25_score(10, 1.5, b)
    print(f"{b:<8.2f} {score:<10.4f}")
```

```python
# 같은 컬렉션에서 BM25 와 TF-IDF 비교

from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

# TF-IDF
vectorizer = TfidfVectorizer(lowercase=True)
tfidf_matrix = vectorizer.fit_transform(docs)
q_vec_tfidf = vectorizer.transform([query])
tfidf_scores = cosine_similarity(q_vec_tfidf, tfidf_matrix).flatten()

# BM25
bm25 = BM25Retriever(docs)
bm25_results = bm25.retrieve(query, k=len(docs))
bm25_scores = {doc_idx: score for doc_idx, score in bm25_results}

print("\nBM25 vs TF-IDF comparison:")
print(f"{'Doc':<40} {'TF-IDF':<10} {'BM25':<10} {'Diff':<10}")
for idx in range(len(docs)):
    doc_display = (docs[idx][:35] + "...") if len(docs[idx]) > 35 else docs[idx]
    tfidf_score = tfidf_scores[idx]
    bm25_score = bm25_scores.get(idx, 0)
    diff = abs(tfidf_score - bm25_score)
    print(f"{doc_display:<40} {tfidf_score:<10.4f} {bm25_score:<10.4f} {diff:<10.4f}")
```

## ⚖️ 가정과 한계

- **Eliteness binary assumption**: 실제로 document-term relevance 는 graded (0~1 continuous)
- **Term independence**: 각 term 의 가중치가 독립적 — query term 간 interaction 없음.
- **Static IDF**: Collection 변화 시 IDF 재계산 필요.
- **Parameter tuning**: $k_1, b$ 의 optimal 값은 domain-dependent. 일반적 default 는 모든 도메인에서 최적이 아님.
- **No semantic similarity**: "car" 와 "automobile" 은 다르게 취급.
- **Doesn't scale to very large vocabularies**: Computational overhead 는 sparse representation 이지만 여전히 있음.

## 📌 핵심 정리

$$\boxed{\text{BM25}(d,q) = \sum_{t \in q} \text{IDF}(t) \cdot \frac{\text{TF}(t,d) \cdot (k_1 + 1)}{\text{TF}(t,d) + k_1(1-b+b \cdot |d|/\text{avgdl})}}$$

| 구성 | 역할 | 기본값 |
|-----|------|-------|
| $\text{IDF}(t)$ | Term distinctiveness | $\log((N-df+0.5)/(df+0.5))$ |
| $\frac{x(k_1+1)}{x+k_1}$ | Saturation (TF 에 diminishing return) | $k_1=1.5$ |
| $1-b+b\|d\|/\text{avgdl}$ | Length normalization | $b=0.75$ |

> **핵심**: BM25 는 Probabilistic Relevance Framework 에 기초한 엄격한 모델. 30년 이상 강력한 이유는 이론과 경험의 완벽한 균형. 비록 dense 가 semantic 을 더 잘 포착하지만, zero-shot 과 interpretability 에서는 여전히 BM25 를 이기기 어려움. Modern RAG 는 두 가지를 hybrid 로 사용.

## 🤔 생각해볼 문제 (+ 해설)

**문제 1 (기초)**: 컬렉션 1000개 문서, 평균 길이 50 term. "deep" 은 50개 문서에만 나타남. 어떤 문서에서 "deep" 이 5번. BM25 score (k1=1.5, b=0.75)?

<details>
<summary>해설</summary>

$$\text{IDF}(\text{deep}) = \log\left(\frac{1000 - 50 + 0.5}{50 + 0.5}\right) = \log\left(\frac{950.5}{50.5}\right) \approx \log(18.8) \approx 2.93$$

$$\text{TF}=5, k_1=1.5, b=0.75, |d|=50, \text{avgdl}=50$$

$$\text{BM25} = 2.93 \times \frac{5 \times 2.5}{5 + 1.5 \times (1 - 0.75 + 0.75 \times 50/50)}$$

$$= 2.93 \times \frac{12.5}{5 + 1.5} = 2.93 \times \frac{12.5}{6.5} = 2.93 \times 1.923 \approx 5.63$$

(Comparison: TF-IDF 라면 대략 $5 \times 2.93 = 14.65$로 훨씬 큼 — BM25 의 saturation 효과)
</details>

**문제 2 (심화)**: $k_1=\infty$ 일 때 BM25 의 TF 항이 어떻게 되는가?

<details>
<summary>해설</summary>

$$\lim_{k_1 \to \infty} \frac{\text{TF}(k_1+1)}{\text{TF}+k_1(1-b+b|d|/\text{avgdl})} = \lim_{k_1 \to \infty} \frac{\text{TF} \cdot k_1}{\text{TF}+k_1} = \lim_{k_1 \to \infty} \frac{\text{TF}}{1 + k_1/\text{TF}} = \text{TF}$$

즉, $k_1 \to \infty$ 이면 BM25 → TF (saturation 제거, linear TF 복원). 따라서 $k_1$ 은 **얼마나 빨리 saturation 에 도달할 것인가** 를 제어.
</details>

**문제 3 (논문 비평)**: "BM25 는 2-Poisson 모델에서 유도되었지만, 현대 neural retrieval (BERT 기반) 이 더 표현력 있으니 BM25 는 outdated 다" 는 주장에 대해?

<details>
<summary>해설</summary>

반박: BM25 outdated 가 아니라 **complementary**. 이유:

1. **Zero-shot strength**: Dense model 은 fine-tuning 데이터가 있어야 하지만, BM25 는 데이터 없이도 작동.
2. **Hybrid 우월성**: Dense + BM25 hybrid 가 둘 다 단독으로 쓸 때보다 우수. BEIR benchmark 에서 입증.
3. **Interpretability**: Dense embedding 의 각 차원이 무엇을 의미하는지 해석 불가능. BM25 는 각 term 의 기여도 명확.
4. **Domain shift**: Dense model 을 학습한 domain 과 다른 domain 에서는 BM25 가 더 robust (fine-tuning 없이).

현재 best practice: Dense retrieve (recall@100) + BM25 rerank (또는 그 반대) + cross-encoder final rerank.
</details>
